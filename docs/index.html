DProvenanceKit is open source regression testing and reasoning observability for AI agents. Every run becomes a queryable, diffable trace, and CI fails when the reasoning drifts — a dropped tool call, a looping step, a changed execution path.

Observability tools show you what happened. DProvenanceKit fails the PR that caused it. Think pytest for agent reasoning.

# 01 · record → explain → diff

One import. Traces you can diff, not logs you can grep.

LLM tracing without the ceremony: wrap your steps in trace(…), save a golden run, and diff any later run against it. This is the whole loop — straight from the README, unedited:

five_minute_wow.pypython
from dprovenancekit import trace

# 1. Record an execution
with trace("Agent Workflow"):
    with trace("Retrieve Documents"):
        # your retrieval code here
        pass
    with trace("Verify Claims"):
        # your verification code here
        pass

# 2. Save the trace
trace.save("golden_run.sqlite")

# 3. Print a structural explanation
trace.explain()
# --- Execution Trace (b4f8d2…) ---
# ▶ Started Agent Workflow
#   ▶ Started Retrieve Documents
#   ✔ Finished Retrieve Documents
#   ▶ Started Verify Claims
#   ✔ Finished Verify Claims
# ✔ Finished Agent Workflow

# 4. Catch regressions when the logic changes: rerun the (now buggy) workflow,
#    then diff the current run against the saved golden baseline
trace.diff("golden_run.sqlite")
# --- Trace Diff (Golden vs Current) ---
# ❌ Missing step: Verify Claims

Under the hood this powers anomaly detection, CI gating, and visual trace analysis. Works with your stack via adapters, or plain Python.

# 03 · gate in ci

A CI gate for AI agents. Server-less. Exit code 1.

The dprovenancekit gate CLI runs fully local — no hosted backend. Wire it into any pipeline, or drop in the GitHub Action from the Marketplace (a GitLab CI template ships too). On regression it fails the job and posts a sticky PR comment with the diff.

.github/workflows/reasoning-regression.ymlyaml
name: reasoning-regression
on: pull_request

permissions:
  contents: read
  pull-requests: write   # required for the PR comment

jobs:
  gate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      # Produce a SQLite trace database with your golden + candidate runs.
      - name: Record traces
        run: python scripts/record_traces.py   # your script; writes traces.sqlite

      - name: Regression gate
        uses: Therealdk8890/dprovenancekit-action@v1
        with:
          db-path: traces.sqlite
          golden-context: golden        # newest run recorded with this context id
          candidate-context: candidate
          max-level: none               # strict: any divergence fails

Built-in anomaly rules — Tool Drop and Looping — run from a JSON rule registry, locally or on every PR:

.github/dprov-rules.jsonjson
{
  "rules": [
    { "type": "tool_drop", "required_step": "safety_check" },
    { "type": "looping", "step": "web_search", "max_repeats": 5 }
  ]
}

$ dprovenancekit anomalies --db traces.sqlite --rules rules.json — same rules, no Action required.

# 04 · what ships in the box

Not just a library — the surfaces that make regressions actionable.

✓Server-less gate CLI

dprovenancekit gate diffs candidate vs golden from local SQLite and exits non-zero on drift. No backend, no account.

✓GitHub Action + GitLab CI

A drop-in Marketplace Action and a GitLab CI template that fail the PR/MR and comment the diff.

✓Anomaly rules

Tool Drop and Looping detection out of the box, configured through a JSON rule registry.

✓pytest plugin

Prompt and agent regression testing inside the test suite you already run — gate reasoning like any other assertion.

✓Shareable HTML reports

Regression reports you can attach to a PR or hand to a teammate — no server needed to open them.

✓Zero-dependency core

Python 3.9+, zero third-party dependencies in the core, Apache-2.0. pip install dprovenancekit and go.
